SCCM Client
Troubleshooting

This post goes over common client-side SCCM errors and how to troubleshoot them using CMTrace and powerShell scripting. There is a github repository available at the bottom of this post with PowerShell templates and reference documentation.

Software Updates not advertised

  • CodePipeline
  • CodeBuild
  • CloudFormation

Prerequisites

  • The website.json template found in this github repository should be used in CloudFormation to create a S3 bucket which has two unique properties. First, the access control property is set to public read and second, the website hosting property is enabled and configured to hold an index.html document.
  • A sample file named index.html should be uploaded to the appropriate github repository branch.

CI/CD Pipeline phases

  • Source Phase: The action provider for this pipeline was this github repository. Webhooks were used for the change detection method to ensure a test environment would be generated anytime there were changes to the repository.
  • Testing Phase: CodeBuild was used to test whether the index.html file contained the words "Automation for the People!". This was done by using cat index.html | grep "Automation for the People!" (The generated buildspec.yml is available in the repository)
  • Deploy Phase: If the test phase had no errors then the index.html file was deployed to the S3 bucket initially created in the CloudFormation stack.

Cleanup

  • Ensure you delete the CloudFormation stack and the S3 bucket to avoid incurring any additional charges.

Ideas for Imrpovement

  • Further automate the process by provisioning the S3 bucket inside the CodePipeline.
  • Use a configurable TTL to delete the S3 bucket and any other dependencies.
